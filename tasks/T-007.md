## Goal

GitHub Actions guard every push and a tag turns into a draft release with all non-Apple artifacts: server + TUI binaries for macOS/Linux/Windows, the unsigned Windows MSI, a GHCR server image, and SHA256SUMS.

## Acceptance Criteria

- [ ] `.github/workflows/ci.yml`: rust-linux job (libdbus-1-dev, fmt-check, clippy -D warnings + tests with `--exclude nod-desktop` to skip the webkit2gtk stack), macos job (full `cargo test --workspace` + desktop `npm ci && typecheck && test`), optional path-filtered swift job (build-nod-client-ffi.sh first — xcframework is git-ignored)
- [ ] `.github/workflows/release.yml` on tags `v*` + `workflow_dispatch`: server+TUI matrix (aarch64/x86_64-apple-darwin, x86_64-linux-gnu, aarch64-linux-gnu on ubuntu-24.04-arm, x86_64-windows-msvc), `nod-{server,tui}-{tag}-{target}.tar.gz`/`.zip` naming
- [ ] windows-msi job: `npm run tauri build -- --bundles msi` (override required — tauri.conf.json also lists deb/rpm/appimage); no Linux desktop bundles in v1
- [ ] docker job: `ghcr.io/batteryshark/nod-server:{tag}` + `:latest` from existing Dockerfile
- [ ] release job (tag only): artifacts + `SHA256SUMS` attached to a DRAFT release, body from `docs/release-notes/v{ver}.md`
- [ ] One `workflow_dispatch` run validates every matrix leg before any tag
- [ ] CI green on a push to main

## Notes

Non-goals encoded as workflow comments: no Apple app builds in CI (DMG built/signed locally per user decision); no Windows/Linux artifact signing in v1 (unsigned MSI + SmartScreen documentation). Known risks: Windows cargo build of nod-server is the least-exercised leg (cut from matrix if it misbehaves — desktop client is the Windows story); ubuntu-24.04-arm availability (drop linux-aarch64 if flaky); tauri's WiX download occasionally flaky (retry).
