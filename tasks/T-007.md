## Goal

GitHub Actions guard every push and a tag turns into a draft release with all non-Apple artifacts: server + TUI binaries for macOS/Linux/Windows, the unsigned Windows MSI, a GHCR server image, and SHA256SUMS.

## Acceptance Criteria

- [x] `.github/workflows/ci.yml`: rust-linux job (libdbus-1-dev, fmt-check, clippy -D warnings + tests with `--exclude nod-desktop` to skip the webkit2gtk stack), macos job (full `cargo test --workspace` + desktop `npm ci && typecheck && test`), optional path-filtered swift job (build-nod-client-ffi.sh first — xcframework is git-ignored)
- [x] `.github/workflows/release.yml` on tags `v*` + `workflow_dispatch`: server+TUI matrix (aarch64/x86_64-apple-darwin, x86_64-linux-gnu, aarch64-linux-gnu on ubuntu-24.04-arm, x86_64-windows-msvc), `nod-{server,tui}-{tag}-{target}.tar.gz`/`.zip` naming
- [x] windows-msi job: `npm run tauri build -- --bundles msi` (override required — tauri.conf.json also lists deb/rpm/appimage); no Linux desktop bundles in v1
- [x] docker job: `ghcr.io/batteryshark/nod-server:{tag}` + `:latest` from existing Dockerfile
- [x] release job (tag only): artifacts + `SHA256SUMS` attached to a DRAFT release, body from `docs/release-notes/v{ver}.md`
- [x] One `workflow_dispatch` run validates every matrix leg before any tag
- [x] CI green on a push to main

## Notes

Non-goals encoded as workflow comments: no Apple app builds in CI (DMG built/signed locally per user decision); no Windows/Linux artifact signing in v1 (unsigned MSI + SmartScreen documentation). Known risks: Windows cargo build of nod-server is the least-exercised leg (cut from matrix if it misbehaves — desktop client is the Windows story); ubuntu-24.04-arm availability (drop linux-aarch64 if flaky); tauri's WiX download occasionally flaky (retry).

## Progress

2026-06-10: Done. ci.yml + release.yml live and validated the hard way — the first runs surfaced FOUR real bugs that local development could never catch: (1) the Windows toast handler in nod-desktop had never compiled (E0282 + missing as_ref/Interface — only a Windows build sees cfg(windows) code); (2) appattest's openssl broke the windows-msvc and mac-x86_64 cross builds → release builds now use a vendored-openssl feature (statically built, nasm on MSVC), making every release binary independent of system libssl; (3) BOTH git-ignored TLS fixture sets (relay mtls + nod-server relay-tls) were absent on runners → their generate scripts run before tests; (4) Xcode 16.4 strict concurrency flagged two Sendable violations (DCAppAttestService sent across an await; the menu-bar NSImage static) → fixed properly. Second release dispatch: all 7 jobs green (5-target server+TUI matrix, 1.0.0-versioned MSI, GHCR image build). CI run 27265462229: rust-linux + macos + swift all green. Artifacts verified by downloading the dispatch MSI + mac tarballs and running the deploy.md Path A walk with them (T-009).
