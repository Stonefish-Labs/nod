## Goal

The Windows client verified end-to-end using the actual CI-built MSI in a Windows VM on the Mac, with SmartScreen behavior documented and the tray icon visually confirmed for the first time.

## Acceptance Criteria

- [ ] MSI sourced from a release.yml `workflow_dispatch` run (bits-identical to the release pipeline)
- [ ] VM reaches host server (`http://<host-ip>:8767/health` from VM browser; Parallels/VMware shared net → host LAN IP, UTM → 10.0.2.2)
- [ ] SmartScreen "More info → Run anyway" flow screenshotted (feeds deploy.md + release notes); `Get-FileHash` matches published SHA-256
- [ ] Windows tray nod-head icon visually verified — light/dark taskbar, tooltip, Show/Quit menu (first time seen; macOS half already user-verified)
- [ ] Enroll → receive request (issuer.py from host) → Windows toast → approve-with-text → reject → verified on server
- [ ] Persistence: app relaunch keeps enrollment (Credential Manager); VM reboot keeps enrollment; autostart behavior checked
- [ ] WS reconnect on server restart; uninstall removes tray; reinstall works
- [ ] Windows section of `docs/release-checklist.md` complete; screenshots in `docs/images/`
