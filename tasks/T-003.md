## Goal

Server is release-ready: legacy shims removed, the binary self-contained (embedded admin.html with a dev-time file override), and a reusable end-to-end smoke test that exercises the full enroll → request → WebSocket sync → signed decision loop against a real listener (the only WS coverage in the suite) and can also point at any deployed instance.

## Acceptance Criteria

- [ ] `ensure_column`/`column_exists` helpers, both call sites, and the emoji backfill UPDATE deleted from `server/nod-server/crates/nod-server/src/db/connection.rs` (dead legacy — schema.sql already has both columns)
- [ ] `/admin` serves an `include_str!`-embedded admin.html by default; `NOD_ADMIN_HTML_PATH` env var reads the file per-request for live-edit dev workflow; Dockerfile no longer copies assets
- [ ] New `tests/e2e_smoke.rs`: health poll → admin creates user/channel/subscription/enrollment-code/issuer-token → enroll fake device (P-256) → open `/api/v1/sync` WS → create request via issuer token → WS envelope arrives and per-user projection matches `GET /api/v1/requests` → sign + submit decision → WS resolution + API shows resolved with signed decision
- [ ] Smoke test runs in-process by default (port 0, tempdir) AND against a deployment via `#[ignore]` + `NOD_SMOKE_URL`/`NOD_SMOKE_ADMIN_TOKEN`
- [ ] `scripts/nod-smoke [URL TOKEN]` wrapper; documented in server README Verify section
