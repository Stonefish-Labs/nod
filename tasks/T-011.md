## Goal

A `scripts/release-macos` pipeline that turns the working tree into a Developer ID-signed, notarized, stapled, checksummed `Nod-1.0.0.dmg` that passes Gatekeeper clean on first launch.

## Acceptance Criteria

- [ ] Entitlement risk resolved FIRST: `NodMac.entitlements` carries the restricted App Attest entitlement — throwaway codesign+notarize attempt early; strip from the Mac target if not load-bearing for macOS enrollment (check server device_attestation mac path); escalate to user only if load-bearing
- [ ] One-time setup documented in `docs/APPLE.md`: Developer ID Application cert + `xcrun notarytool store-credentials nod-notary`
- [ ] `client/nod-apple/scripts/release-macos`: build via `build-macos-app` with `NOD_MAC_MARKETING_VERSION=1.0.0` → resolve Developer ID identity (fail loudly) → deep-sign inside-out (`--timestamp --options runtime --entitlements`) → notarize zip (`--wait`, print log on failure) → staple app → `hdiutil` DMG with /Applications symlink → sign + notarize + staple the DMG itself → `spctl --assess` app + DMG → `codesign --verify --deep --strict` → SHA-256
- [ ] `spctl` reports "Notarized Developer ID"; DMG installs and first-launches with no Gatekeeper override
- [ ] macOS section of `docs/release-checklist.md` run from the stapled DMG: enroll, notify, decide, restart persistence

## Notes

`build-macos-app` stays the dev build (Apple Development / ad-hoc fallback). The current Apple Development DMG is a testing artifact only — AGENTS.md explicitly requires Developer ID + notarytool + staple for the published release. Known footgun: rebuild the FFI xcframework if nod-client-core changed (build-macos-app handles this).
