Nod is a self-hosted interrupt and decision surface for personal agents,
automations, and services: software creates a structured request, Nod routes
it to the right people across their devices, and the issuer gets back a
cryptographically signed, durable decision record. This is the first public
release.

## What's in this release

## Getting started

[docs/deploy.md](https://github.com/batteryshark/nod/blob/v1.0.0/docs/deploy.md)
walks through running the server on a regular machine — download, set
`NOD_ADMIN_TOKEN`, open the admin panel, enroll a client. Five minutes,
no Docker required.

## Signing status

- **macOS app**: Developer ID signed and notarized — Gatekeeper passes on
  first launch.
- **Windows desktop app**: a zipped exe (no installer — put it wherever you
  want), unsigned in this release. SmartScreen will show "Windows protected
  your PC" on first run — choose **More info → Run anyway** after verifying
  the download: `Get-FileHash <file>` must match `SHA256SUMS`.
- **Server/TUI binaries**: unsigned tarballs/zips; verify with
  `shasum -a 256 -c` against `SHA256SUMS`. On macOS, clear quarantine with
  `xattr -d com.apple.quarantine <binary>`.

## Security notes

- Decisions are signed on-device with P-256 keys registered at enrollment —
  Secure Enclave on Apple hardware, software keys elsewhere — and verified
  against the immutable request snapshot, with per-key nonce replay
  rejection.
- The server is meant to stay private: bind it to localhost or a tailnet and
  front it with Tailscale Serve or a TLS reverse proxy.
- Apple push (optional, iPhone background delivery) supports an in-process
  route or an mTLS relay that keeps APNs credentials out of the main server.

## Known limitations

- Linux desktop bundles are not part of this release (the TUI and server
  cover Linux); they'll ship once there's a tested path.
- iOS is TestFlight-only.
- Windows binaries are unsigned (no code-signing certificate yet), and
  the desktop app intentionally ships as a bare exe rather than an installer.
